# GEMM Optimization

This repository contains a CUDA learning example for optimizing single-precision general matrix multiplication (GEMM):

```text
C = A x B
A: M x K
B: K x N
C: M x N
```

The current code uses row-major matrix storage and compares each GPU kernel against a reference result generated by the warmup/native kernel path.

## Implemented Kernels

- `gemmnative`: one CUDA thread computes one output element.
- `gemmUnroll2`: one thread computes two output columns.
- `gemmUnroll4`: one thread computes four output columns.
- `gemmSmem`: tiled GEMM using shared memory.
- `gemmSmemPad`: shared-memory GEMM with padding.
- `gemmSmemUnroll2`: shared-memory GEMM combined with column-direction unrolling.
- `gemmSmemUnroll4`: shared-memory GEMM combined with four-column unrolling.

## Build

Compile with `nvcc`:

```bash
nvcc -arch=sm_89 GEMM.cu -o GEMM
```

For another GPU architecture, replace `sm_89` with the architecture supported by your device.

## Run

```bash
./GEMM
```

Optional block dimensions can be passed as command-line arguments:

```bash
./GEMM 16 16
```

The default matrix size in the source is:

```cpp
int M = 1 << 12;
int N = 1 << 12;
int K = 1 << 12;
```

## Validation

The program checks each optimized kernel against a reference result with a tolerance of `1e-2`, which is used to account for normal floating-point differences between CPU/GPU and different CUDA execution paths.

Expected successful output includes:

```text
Match !
```

for each kernel being validated.

## Profiling

The kernels can be profiled with NVIDIA Nsight Compute or Nsight Systems. A typical optimization progression in this file is:

```text
native -> unroll2 -> unroll4 -> shared memory -> shared memory + unroll2 -> shared memory + unroll4
```

One profiling run on an NVIDIA GeForce RTX 4060 Laptop GPU produced the following kernel durations for `M = N = K = 4096` and `16 x 16` thread blocks:

| Kernel | Duration |
| --- | ---: |
| `gemmnative` | 248.45 ms |
| `gemmUnroll2` | 186.59 ms |
| `gemmUnroll4` | 177.22 ms |
| `gemmSmem` | 177.99 ms |
| `gemmSmemPad` | 263.77 ms |
| `gemmSmemUnroll2` | 158.54 ms |
| `gemmSmemUnroll4` | 149.35 ms |

In this version, `gemmSmemUnroll4` is the fastest kernel in the benchmark above, reaching about `1.66x` speedup over the native kernel.
